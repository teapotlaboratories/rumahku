# M3 — On-device seeding (the quality lever)

Goal: seed on-device Gaussian-splat training with a real point cloud captured by
ARCore, instead of random init — so reconstructions are actually usable within a
phone-reasonable iteration budget. No COLMAP, no cloud.

Context: `docs/PHASE2.md` (Phase 2 plan), `docs/M1.md` (the on-device trainer),
`docs/worklog/2026-07-07.md` (M0/M1 results + the seeding validation below).

---

## 1. Why this is the priority

On-device, random init is not merely worse — it's **unusable** at a phone-time
iteration budget. Measured on the Pixel 6 (Mali-G78), same COLMAP poses, same
1000 iters @720px, only the init differs:

| Init | PSNR | SSIM | Visual |
|---|---|---|---|
| **Seeded** (COLMAP points) | **21.5** | **0.82** | recognizable room |
| Random | 10.7 | 0.40 | foggy blob |

**+10.8 dB from seeding** — larger than the off-device +6.75 dB (which was at 30k
iters; at ~1000 iters random init is nowhere near converged). Seeding reaches a
legible reconstruction in ~3 min on the phone GPU. → seeding is essential.

---

## 2. How Brush consumes a seed (no FFI/core changes)

Brush's nerfstudio loader reads a top-level `"ply_file_path"` from
`transforms.json`; if present it loads that point-cloud `.ply` as the initial
splats (`brush-dataset` `nerfstudio.rs`). Its PLY importer requires **only
positions** (`x y z`); colors are optional. So M3 is entirely capture-side:
write a `seed.ply` and add one JSON field. (Brush also auto-seeds from COLMAP
`points3D` when given a COLMAP dataset — that's how §1 was measured.)

---

## 3. The seed source: ARCore feature points

ARCore's `Frame.acquirePointCloud()` yields feature points as packed
`[x, y, z, confidence]` in the **same world frame as the camera poses** we write
— so the seed is aligned for free. Points carry stable IDs, so we dedupe across
frames and keep the latest position for each.

(Depth API would give a denser cloud and is a future upgrade; feature points are
universally available and sufficient to prove/ship the lever.)

---

## 4. Implementation (capture-side)

- `capture/SeedPointCloud.kt` — accumulates ARCore points across frames
  (HashMap by id, confidence ≥ 0.3), thread-safe.
- `capture/CaptureSession.kt` — every stable-tracking frame,
  `acquirePointCloud().use { addFrom(ids, points, MIN_CONF) }`; cleared on start;
  snapshot handed to the writer on stop.
- `capture/DatasetWriter.kt` — `finish(intrinsics, seedPoints)` writes an ASCII
  positions PLY `seed.ply` (when ≥ 50 points) and adds `"ply_file_path":
  "seed.ply"` to `transforms.json`.

The reconstruct path (M1) is unchanged — it feeds the capture dir to
`nativeTrain`, and Brush now finds the seed automatically.

---

## 5. Status & follow-ups

- Mechanism validated on-device (+10.8 dB, §1). Capture-side code written and
  compiles.
- **Field test needed:** a real in-app scan produces ARCore points → seed.ply →
  reconstruct, and confirm the quality lift on a live scan (can't be done from
  the bench; needs someone physically scanning a room).
- Follow-ups: **color the seed points** (sample the camera image at projection)
  for better initial color; try the **Depth API** for a denser seed; tune
  confidence threshold / max points.
