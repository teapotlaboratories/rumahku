## Made phone unlock non-flaky
- Root causes of the flaky unlock: (1) phone is in **gesture-nav mode**, so the
  bottom-edge swipe was eaten as a system gesture; (2) digits sent during the
  bouncer raise-animation were dropped; (3) `deviceLocked`/screencap read stale.
- Fix: **`tools/adb-unlock.sh`** — runs an on-device timed sequence (sleeps
  between wake / mid-screen swipe / digits), resets each attempt, retries, and
  polls until unlock settles. Carries NO PIN (reads `$PIXEL_PIN` or
  `~/.config/rumahku/pixel-pin`, mode 600, gitignored). `tools/adb-unlock.sh
  <package>` also launches an app. Tested **4/4 first-attempt** unlock+launch.
  A subtle bug fixed along the way: detection polled `deviceLocked` (which
  settles slower than the real unlock) and each retry re-locked a
  just-unlocked device — now it checks "already unlocked" before re-locking.
  Documented in `docs/BENCH.md`.

## M0 RESULT — PASS ✅ (Brush trains a splat on the Pixel 6 GPU)
Ran `brush-cli` on the Pixel 6 (Mali-G78) against `scan99`:
- **wgpu→Vulkan initialised on Mali** and compiled/autotuned WGSL compute
  kernels — the #1 project risk, now retired on the worst-case GPU.
- Trained to completion and exported a **valid Gaussian splat**: 13,604
  Gaussians, SH degree 3 (`rumahku-splat/m0_pixel6_300.ply`).
- Perf (warm cache, 25 views @ 640px, 300 iters): **54 s → ~5.5 iters/sec**,
  peak RSS **311 MB**, peak temp **32.1 °C**. Zero panics, zero cache errors.
- **Key integration finding:** the adb shell HOME is read-only, so cubecl's
  kernel cache is disabled by default → cold autotune is ~40 s/kernel (brutal).
  Fix: give Brush a **writable HOME/cache dir** (`HOME=/data/local/tmp/home`).
  M1 must hand Brush a writable cache dir or every cold start pays autotune.
- Extrapolation: at ~5.5 it/s a full 30k-iter train ≈ ~90 min on Mali →
  confirms the "coarse-then-refine over minutes" expectation; M3 seeding should
  cut the iters needed. Bigger runs (more views, higher res) still need
  memory/thermal validation (this run was deliberately tiny).
